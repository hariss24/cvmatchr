## Résumé exécutif

**Note globale : B-.** Le projet est un outil personnel/local mature et étonnamment soigné sur la sécurité (anti-SSRF, CSRF, rate-limiting), avec 101 tests qui tournent en 2,6 s — mais il souffre de sa croissance par boucle autonome : deux pipelines parallèles (HTML et JSON) qui dupliquent la logique métier, deux fichiers monolithes (`app.js` : 2 402 lignes, `app.py` : 1 273 lignes), et aucun filet CI.

**Top 3 risques :**
1. Pas de `.dockerignore` alors que le Dockerfile fait `COPY . .` — une image construite localement embarquait le `.env` ✅ corrigé
2. Le schéma JSON (`_normalize_resume`) **perd silencieusement** les sections Projets/Certifications/Bénévolat que le pipeline HTML supporte — violation de la règle n°3 du CLAUDE.md
3. Un test échouait (`test_slug_basic`) sans CI pour le signaler ✅ corrigé

**Top 3 opportunités :**
1. Terminer la parité du schéma JSON (`_normalize_resume`) pour cesser de perdre des données silencieusement
2. CI GitHub Actions (pytest + ruff) — 1 heure de travail, protège la boucle autonome de nuit
3. Extraire `prompts.py` et `scraper.py` de `app.py` (déplacement pur, ~40% de réduction)

## Repo Map

**Objectif :** application web locale (Flask) de création/adaptation de CV et lettres de motivation, avec conversion HTML→PDF (Playwright/Chromium), adaptation IA (Gemini/Anthropic), score ATS, scraping d'offres d'emploi, et intégration MCP pour Claude Desktop.

**Stack :** Python 3.14, Flask, Playwright (fallback WeasyPrint), google-genai + anthropic, SQLite/MongoDB, frontend vanilla JS + Monaco Editor, IndexedDB/localStorage côté navigateur.

| Fichier | Rôle | Lignes |
|---|---|---|
| `app.py` | Routes Flask + auth + CSRF + ~350 l. prompts IA + scraping + lanceur Tkinter | 1 273 |
| `ai_engine.py` | Appels Gemini/Anthropic : streaming, chat, tailoring, ATS, pack, OCR PDF→JSON | 755 |
| `pdf_engine.py` | HTML→PDF (Playwright/WeasyPrint), whitelists, anti-SSRF | 128 |
| `archive.py` | Historique SQLite/MongoDB, sanitisation des noms de fichiers | 359 |
| `quota.py` | Quota journalier en mémoire (50/jour) | 36 |
| `mcp_server.py` | Serveur FastMCP pour Claude Desktop | 109 |
| `static/js/app.js` | Le "mastodonte" frontend : Monaco, chat IA, ATS, snapshots, IndexedDB | 2 402 |
| `static/js/resume-form.js` | Mode Formulaire : JSON → rendu HTML | 500 |
| `tests/` | Pytest, mocks IA, bonne couverture des endpoints | 101 tests |

**Architecture clé :**
- Le frontend JS est le moteur de rendu HTML (JSON → HTML → envoie au `/convert`)
- `/convert` génère le PDF et renvoie les bytes ; il ne persiste **plus** rien côté serveur
- L'historique vit dans IndexedDB (navigateur) ; le backend SQLite ne sert que pour le MCP
- Migration en cours : JSON est la nouvelle source de vérité, HTML n'est plus que le rendu final

## Audit — Findings

### 🔐 Sécurité

**S1 — HIGH ✅ Corrigé (dockerignore) : clé API dans `.env` + pas de `.dockerignore` + `COPY . .`**  
`.env` contient une clé Gemini réelle. Sans `.dockerignore`, toute image Docker construite localement embarquait la clé.  
→ `.dockerignore` créé lors des quick wins.

**S2 — MEDIUM : URLs d'offres envoyées à r.jina.ai sans le dire**  
`app.py:1096` : quand le scraping Playwright est bloqué, l'URL fournie est rejouée via `https://r.jina.ai/{url}`. Le README:104 affirme « your HTML never leaves your machine ». Décision : priorité basse (mode remote non prioritaire).

**S3 — MEDIUM : rate-limit de login inopérant derrière un proxy (Render)**  
`app.py:114` utilise `request.remote_addr`. Sans `ProxyFix`, toutes les requêtes partagent l'IP du load balancer. Décision : priorité basse (remote non prioritaire).

**S4 — LOW : `SECRET_KEY` aléatoire par processus**  
`app.py:54` : sans `SECRET_KEY` en env, chaque redémarrage invalide les sessions et jetons CSRF. Décision : priorité basse.

**S5 — LOW : le quota est consommé même si l'appel IA échoue ensuite**  
`app.py:631`. À corriger un jour (M3.4).

### 🏗️ Architecture & correction des données

**A1 — HIGH ⚠️ RESTE À FAIRE : schéma JSON perd des sections**  
Le squelette HTML (`app.py:348-397`) a des sections **Projets, Certifications, Bénévolat**.  
Le schéma JSON (`_RESUME_SCHEMA_DESC`, `ai_engine.py:500`) et `_normalize_resume` (`ai_engine.py:519`) ne connaissent que : name, title, summary, experience, education, skills, languages, interests.  
**Conséquence :** un CV importé via `/api/pdf-to-resume` ou adapté via `/api/tailor-resume` **perd silencieusement** ses projets et certifications. Violation directe de la règle n°3 du CLAUDE.md.

**A2 — HIGH : double pipeline HTML/JSON, logique dupliquée**  
Les 4 niveaux de tailoring existent en deux exemplaires :
- `_TAILOR_SYSTEMS` (`app.py:501-582`) — prompts HTML, utilisés par le mode Expert
- `_RESUME_TAILOR_RULES` (`ai_engine.py:640-674`) — prompts JSON, utilisés par le mode Formulaire

**Important :** `/api/tailor` (HTML) est encore actif et nécessaire — le mode Expert (app.js ~ligne 2264) l'utilise encore, y compris le mode CV Maître (`is_master`). Ne **pas** supprimer. La duplication est le prix du support des deux modes.

**A3 — MEDIUM : `app.py` est un fourre-tout**  
Routes Flask + ~350 lignes de prompts + moteur de scraping (137 lignes, `_scrape_job_text`) + lanceur Tkinter. Quatre responsabilités sans frontière. Solution : extraire `prompts.py` et `scraper.py` (déplacement pur).

**A4 — MEDIUM : `app.js` = 2 402 lignes, ~193 fonctions, zéro test**  
Refactoring différé (décision du 9 juin, confirmée) — à ne faire qu'après avoir des tests de caractérisation.

## Stratégie

### Thème 1 — Parité du schéma JSON (prioritaire)
Ajouter `projects`, `certifications`, `volunteer` à `_normalize_resume`, aux prompts JSON, et au rendu `resume-form.js`. Une seule source de vérité = une seule implémentation des règles métier.

### Thème 2 — CI avant tout le reste
GitHub Actions (pytest + ruff) en moins d'une heure. Prérequis de la boucle autonome.

### Thème 3 — Dégrossir les monolithes, sans big-bang
Extraire `prompts.py` (constantes) et `scraper.py` (`_scrape_job_text` + helpers SSRF) depuis `app.py`. Déplacement pur, ~40% de réduction. Ne pas toucher à `app.js` avant d'avoir des tests.

### Ce qu'on NE fait PAS
- Supprimer `/api/tailor` HTML — encore utilisé par le mode Expert
- Réutiliser Playwright entre requêtes (pas utile pour un usage mono-utilisateur local)
- Remplacer Tkinter, ajouter bundler JS, TypeScript, durcissement MongoDB
